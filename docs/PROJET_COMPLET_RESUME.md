### 5. Scripts Python

#### `extract_pdf_images_simple.py`
**Rôle** : Extraire toutes les images du PDF

**Résultat** : 52 images extraites avec succès

**Snippet** :
```python
import fitz  # PyMuPDF

PDF_PATH = r"C:\Users\User\Dropbox\PRAEDIUM TECH- CONTROLLED DOCUMENT\PLAQUETTE FINALE.pdf"
OUTPUT_DIR = r"Frontend\src\assets\images\branding"

pdf = fitz.open(PDF_PATH)
for page_num in range(len(pdf)):
    page = pdf[page_num]
    images = page.get_images(full=True)

    for img_index, img in enumerate(images):
        xref = img[0]
        base_image = pdf.extract_image(xref)
        image_bytes = base_image["image"]
        image_ext = base_image["ext"]

        filename = f"page_{page_num + 1:02d}_img_{img_index + 1}.{image_ext}"
        filepath = os.path.join(OUTPUT_DIR, filename)

        with open(filepath, "wb") as f:
            f.write(image_bytes)
```

## 🐛 Problèmes Rencontrés et Solutions

### Problème 1 : Encodage Unicode dans Script Python
**Erreur** :
```
UnicodeEncodeError: 'charmap' codec can't encode character '\U0001f3a8'
```

**Cause** : Émojis incompatibles avec l'encodage console Windows

**Solution** : Remplacement de tous les émojis par équivalents ASCII
- 🎨 → [*]
- ✅ → [OK]
- ❌ → [ERROR]
- 💾 → [SAVE]

### Problème 2 : Import PyMuPDF Échoué
**Erreur** :
```
ImportError: No module named 'fitz'
```

**Cause** : PyMuPDF non installé dans l'environnement Python actif

**Tentative 1** : `pip install PyMuPDF --quiet` (échoué)

**Solution** : Installation dans l'environnement correct
```bash
python -m pip install pymupdf
```

**Résultat** : Import réussi, extraction fonctionnelle

### Problème 3 : Port 5173 Déjà Utilisé
**Erreur** : Port par défaut Vite occupé

**Solution** : Vite a automatiquement basculé sur le port 5174
- URL finale : http://localhost:5174/
